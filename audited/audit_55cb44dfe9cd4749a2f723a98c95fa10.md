Audit Report

## Title
BLS Signature Aggregation Denial-of-Service: Malformed Validator Signature Prevents QC/TC Formation

## Summary
A malicious validator can submit a malformed (non-subgroup-valid) BLS signature that, when included in the aggregation set, causes BLS signature aggregation to fail hard. Due to the Aptos consensus code structure, such aggregation failures are never recovered by filtering out invalid signatures. As a result, a single faulty/malicious validator can cause permanent liveness failure (no QC or TC can be formed) for a round, breaking consensus liveness, even if >1/3 honest quorum exists.

## Finding Description
Within the Aptos consensus system, votes are collected via `PendingVotes::insert_vote`. Individual validator signatures are **not** verified at insert time. When quorum voting power is reached, the code attempts to aggregate all signatures via BLS (see `aggregate_signatures`). If any included signature is malformed (e.g., not a valid group element, or otherwise purposely malformed), the BLS aggregate call fails.

In this case, the error is propagated up the stack, and no quorum certificate (QC) or timeout certificate (TC) is formed. Crucially: the code never attempts to filter out *which* signature caused the aggregation to fail. This means any single malicious validator can permanently block QC/TC formation, stalling consensus.

This breaks liveness under <1/3 byzantine assumption and allows an adversary to violate critical protocol invariants.

## Impact Explanation
**Impact: Critical**

- A single malicious validator can stall QC/TC formation by injecting a malformed signature.
- Permanent DoS on the affected round (and possibly onward progress) without requiring >1/3 malicious voting power.
- Breaks "Total loss of liveness/network availability" in the bug bounty categories.
- Validators/Nodes halt or fail to make progress.

## Likelihood Explanation
This attack requires a malicious validator (any single validator, regardless of stake) to submit one invalid BLS signature. The attack is trivial to execute by modifying a validator client node to submit a purposely invalid signature. No validator collusion or byzantine majority is required.

## Recommendation
On aggregation/verification failure, before returning an error, the signature set must be iteratively filtered to remove invalid signatures; only then should aggregation be retried (as in the fallback for verification failure already present elsewhere in the code).

Suggested fix: always attempt to filter and exclude invalid signatures if any part of signature aggregation/verification fails, before giving up on QC/TC construction.

## Proof of Concept

1. Deploy a validator with a modified client that, on vote, produces a malformed/non-subgroup-checked BLS signature.
2. Observe that when this validator's vote is included in the vote set, QC/TC aggregation fails and is not retried with the faulty vote excluded.
3. The round cannot reach QC or TC, permanently stalling consensus for all nodes.

    pub fn aggregate_and_verify(
        &mut self,
        verifier: &ValidatorVerifier,
    ) -> Result<(T, AggregateSignature), VerifyError> {
        let aggregated_sig = self.try_aggregate(verifier)?;

        match verifier.verify_multi_signatures(&self.data, &aggregated_sig) {
            Ok(_) => {
                // We are not marking all the signatures as "verified" here, as two malicious
                // voters can collude and create a valid aggregated signature.
                Ok((self.data.clone(), aggregated_sig))
            },
            Err(_) => {
                self.filter_invalid_signatures(verifier);

                let aggregated_sig = self.try_aggregate(verifier)?;
                Ok((self.data.clone(), aggregated_sig))
            },
        }
    }

    pub fn aggregate_signatures<'a>(
        &self,
        signatures: impl Iterator<Item = (&'a AccountAddress, &'a bls12381::Signature)>,
    ) -> Result<AggregateSignature, VerifyError> {
        let mut sigs = vec![];
        let mut masks = BitVec::with_num_bits(self.len() as u16);
        for (addr, sig) in signatures {
            let index = *self
                .address_to_validator_index
                .get(addr)
                .ok_or(VerifyError::UnknownAuthor)?;
            masks.set(index as u16);
            sigs.push(sig.clone());
        }
        // Perform an optimistic aggregation of the signatures without verification.
        let aggregated_sig = bls12381::Signature::aggregate(sigs)
            .map_err(|_| VerifyError::FailedToAggregateSignature)?;

        Ok(AggregateSignature::new(masks, Some(aggregated_sig)))
    }
